① 什么是 Azure Databricks Genie

Genie 是构建在 Unity Catalog 之上的 AI/BI 自然语言接口。业务用户用中文 / 英文向 Lakehouse 提问，Genie 自动生成 SQL、查询数据、并给出可视化与解释。

🎯 不是什么

  • ❌ 不是"ChatGPT 直接接数据库"
  • ❌ 不是绕过权限的 LLM 工具
  • ❌ 不是只会简单聚合的玩具

✅ 是什么

  • 继承 Unity Catalog 全部治理（权限 / Lineage / 审计）
  • 由数据团队"喂养"业务术语 + 示例 + 指标定义
  • SQL 完全透明 + 可解释 + 可点踩反馈
  • 提供 REST API，可嵌入任意业务系统

现场观察点（最值得说的 3 件事）

1. 治理是真的

切换 Persona，同一问题、同一 SQL，但 UC 在执行时透明附加过滤与脱敏。这不是 App 层兜底，是从 SQL Editor、Power BI 连过来都一致的"单一治理面"。

2. 指标是锁的

sae_rate 不是 Genie 现编——是数据团队在 UC Metric View 里一次定义、永久复用。Genie 只决定从哪个维度切片，业务用户随便问，口径绝不会被改。

3. 可被嵌入

公网 demo 本身就是证明：Genie 是 API-first 产品。可嵌入医院门户、CRM、EDC，让医生 / CRA / PM 在自己熟悉的系统里直接问数据。

这三点，正是 Genie 与"普通 LLM 接数据库工具"的本质区别。
